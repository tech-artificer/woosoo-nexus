# Woosoo Nexus — Design Alignment Handoff

**Read this first.** It is the single front door to the bundle — a developer who was not part of the design conversation can implement the entire alignment from here alone.

- **Project:** Woosoo Nexus admin panel
- **Stack:** Laravel 12 + Vue 3 (Inertia) + Tailwind + shadcn/ui
- **Scope:** Visual alignment of the admin UI to the approved Woosoo brand system
- **Guarantee:** Visual-only. No business logic, routing, API, or data-model changes in any part of this bundle.
- **Generated by:** Woosoo Design · May 2026

---

## 1. What this bundle is

The admin already has the correct brand tokens defined in `app.css` (amber `#F6B56D`, Raleway / Kanit / JetBrains Mono, dark sidebar, light + dark themes). What diverged is how components and pages *consume* those tokens. This bundle corrects that, **outside-in**, in four ordered steps plus a page-by-page tail.

There are **two kinds of file** in this bundle. Do not confuse them:

| Kind | Where | What to do with it |
|---|---|---|
| **Reference files** — real, production-ready `.vue` / `.css` / `.ts` | `step-1/` … `step-4/` (everything except `AGENT_INSTRUCTIONS.md`) | **Copy into the codebase** at the mapped destination, then rebuild. These are drop-in replacements, not mockups. |
| **Visual specs** — HTML documents | `specs/` | **Reference only.** Open in a browser to see the intended before/after, exact diffs, and rationale. Never ship these to the app. |

> The HTML in `specs/` is a *spec*, not source. The Vue/CSS/TS in the step folders is the actual code to ship.

---

## 2. Fidelity

**High-fidelity.** Every change is a final, exact value — hex codes, radii, class strings, and CVA variants are all production values, not approximations. Apply them literally; do not round or "improve".

---

## 3. How to apply — order matters

Apply the steps **in sequence**. Later steps assume earlier ones are in place (e.g. Step 4 deliberately drops a redundant `border-black/8` because Step 3's `Card.vue` already supplies it).

Each step folder has its own `AGENT_INSTRUCTIONS.md` with per-file destinations, a backup command, and a QA checklist. The consolidated copy-and-build script lives in **`README.md`** (§ *Quick-apply all steps*). In short:

1. Back up the files you are about to replace (`*.bak`).
2. Copy each step's reference files to their destinations (table in each `AGENT_INSTRUCTIONS.md`).
3. Rename the two prefixed files on copy: `*-index.ts → index.ts` (Step 3) and `*_Index.vue → Index.vue` (Step 4).
4. After the last step: `npm run build` and `php artisan view:clear && php artisan cache:clear`.

## 5. Beyond the four steps

The four steps cover the shell, the shared primitives, and the three highest-traffic pages. The remaining ~24 pages (Reports suite, POS, Service Requests, Menus, Users, Branches, Roles, Monitoring, Event Logs, Settings, …) are documented in **`IMPLEMENTATION_HANDOVER.md`**.

That file is a per-page change log of **inline class-string edits** — there are no full-file replacements for those pages because each change is a small swap (raw Tailwind color → brand token, stale `p-6` wrapper → hero card, `font-bold` → `font-semibold`). It includes copy-paste structural patterns (hero card, table card, status pill, brand table rows), a priority/implementation order, and a full-sweep QA checklist. Tackle it after Steps 1–4 are live.

## 6. Design tokens (reference)

Already defined in `app.css` — consume, don't redefine.

### Color
| Token | Value | Use |
|---|---|---|
| `woosoo-accent` | `#F6B56D` | Amber — primary actions, active states, badges |
| `woosoo-primary-dark` | `#B08047` | Amber dark — hover, warning text |
| `woosoo-primary-light` | `#FCD8BA` | Amber light — subtle tints |
| `woosoo-dark-gray` | `#252525` | Text on amber backgrounds |
| `woosoo-green` | `#16A34A` | Success / live status |
| `woosoo-blue` | `#2563EB` | Info / accent states |
| `woosoo-red` / `destructive` | `#DC2626` | Errors, destructive actions |
| Sidebar active bg | `#2a1e0c` | Active nav item background |
| Sidebar active border | `#3d2c14` | Active nav item border |

### Raw Tailwind colors to never use — replace with the tokens above
`emerald-* → woosoo-green` · `yellow-* / amber-* → woosoo-accent / woosoo-primary-dark` · `blue-600 → woosoo-blue` · `rose-* → destructive` · `purple-* / indigo-* → woosoo-blue`

### Type
| Token | Family | Use |
|---|---|---|
| `--font-header` | Raleway | Headings, buttons, labels (weights 400–800) |
| `--font-sans` | Kanit | Body text, table content |
| `--font-mono` | JetBrains Mono | IDs, numbers, code |

### Radius & shape
- Brand `--radius`: **0.5rem** (`rounded-lg`) for inputs, buttons, sidebar items, tabs
- Page content / hero / table cards: **`rounded-[26px]`**
- Inner stat cards: **`rounded-[22px]`** / **`rounded-[18px]`**
- Card border: **`border-black/8`** (light) · **`dark:border-white/10`** (dark)
