Woosoo Nexus · Design Alignment Series
Alignment Roadmap
A four-step, low-risk path to bring the real Laravel 12 + Vue 3 admin into pixel alignment with the approved Woosoo brand system. Each step is self-contained, visual-only, and applies in order — no business logic, API, or data-model changes in any step.
Roadmap Step 1 · Look & Feel Step 2 · Layout Step 3 · Components Step 4 · Pages
How the series works
Work outside-in: fix the shell first, then the shared primitives, then individual pages. Each step ships as a folder under handoff/ with replacement files plus an AGENT_INSTRUCTIONS.md — paste those to your dev or coding agent and rebuild.
Visual-only guarantee

Every change is a CSS class string, CVA variant, or template-markup edit. Props, computed values, WebSocket logic, routes, and data flow are untouched in all four steps.

Tokens already correct no work

Brand tokens — amber #F6B56D, Raleway / Kanit / JetBrains Mono, dark sidebar, light & dark themes — are already defined in app.css. The series consumes them; it doesn't redefine them.

Quick-apply all steps
From the project root, copy each step's files into place and rebuild once at the end.
terminal · from project root
# Step 1 — shell
cp handoff/step-1/app.css              resources/css/app.css
cp handoff/step-1/index.ts             resources/js/components/ui/sidebar/index.ts
cp handoff/step-1/NavMain.vue          resources/js/components/NavMain.vue
cp handoff/step-1/AppSidebar.vue       resources/js/components/AppSidebar.vue
cp handoff/step-1/AppSidebarHeader.vue resources/js/components/AppSidebarHeader.vue

# Step 2 — layout
cp handoff/step-2/AppSidebarHeader.vue resources/js/components/AppSidebarHeader.vue
cp handoff/step-2/AppContentLayout.vue resources/js/layouts/AppContentLayout.vue

# Step 3 — primitives  (rename *-index.ts → index.ts)
cp handoff/step-3/badge-index.ts       resources/js/components/ui/badge/index.ts
cp handoff/step-3/button-index.ts      resources/js/components/ui/button/index.ts
cp handoff/step-3/Card.vue             resources/js/components/ui/card/Card.vue
cp handoff/step-3/TabsList.vue         resources/js/components/ui/tabs/TabsList.vue
cp handoff/step-3/TabsTrigger.vue      resources/js/components/ui/tabs/TabsTrigger.vue

# Step 4 — pages  (rename *_Index.vue → Index.vue)
cp handoff/step-4/Dashboard.vue        resources/js/pages/Dashboard.vue
cp handoff/step-4/Orders_Index.vue     resources/js/pages/Orders/Index.vue
cp handoff/step-4/Devices_Index.vue    resources/js/pages/Devices/Index.vue

# Rebuild & clear caches
npm run build
php artisan view:clear && php artisan cache:clear
✓
Apply in order. Steps 2–4 assume the prior steps are in place — e.g. Step 4 drops a redundant border-black/8 because Step 3's Card.vue already supplies it.
