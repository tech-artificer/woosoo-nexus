# Documentation PR Checklist

Use this checklist before merging any documentation change.

- [ ] Does this duplicate an existing canonical document?
- [ ] Does this introduce a second production deployment flow?
- [ ] Does this contradict `compose.yaml` deployment authority?
- [ ] Does this belong in `docs/archive/` instead of active docs?
- [ ] Is this temporary debugging content that should not be canonical?
- [ ] Is this historical/transitional content that should be archived?
- [ ] Does this create ownership ambiguity between repositories?
- [ ] Is `docs/INDEX.md` still accurate after this change?
