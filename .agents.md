# Agent Rules — Woosoo Nexus

**Scope: woosoo-nexus only.** Repo-local docs entrypoint: `docs/INDEX.md`. For ecosystem-wide authority (cross-app contracts, order state, real-time events, agent rules), the single source is the **Authority Map** in the platform repo's `AGENTS.md` (`woosoo-platform/AGENTS.md`).

## Scope

This repo owns:

- Laravel admin/API
- POS/Krypton integration
- Docker/Nginx/Reverb orchestration
- Integrated deployment of sibling `tablet-ordering-pwa`

## Hard Rules

- Do not modify `tablet-ordering-pwa` files from this repo.
- Do not hardcode LAN IPs.
- Do not change POS stored procedure order without explicit approval.
- Do not deploy tablet without verifying sibling repo branch/commit.
- Do not merge dependency PRs into deployment-fix PRs.

## Required Before Merge

```bash
composer test
npm run build
```

For deployment changes:

**PowerShell:**
```powershell
$env:TABLET_DOCKERFILE = "Dockerfile.prod"
bash ./scripts/deployment/verify-tablet-deploy-context.sh
docker compose config
```

**Bash (WSL/Git Bash):**
```bash
export TABLET_DOCKERFILE="Dockerfile.prod"
./scripts/deployment/verify-tablet-deploy-context.sh
docker compose config
```

Note: `verify-tablet-deploy-context.sh` is a bash script. On Windows, invoke it via `bash ./scripts/deployment/verify-tablet-deploy-context.sh` from PowerShell, or run directly in WSL/Git Bash.
