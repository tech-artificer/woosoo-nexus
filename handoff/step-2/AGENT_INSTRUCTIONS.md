## Context

Step 1 established the flat topbar and amber sidebar. Step 2 refines two layout-level files:

1. **Dark mode toggle** — the topbar right side currently shows only "Secure workspace". Step 2 adds a sun/moon button that persists the user's preference to `localStorage` and toggles `.dark` on `<html>`, consistent with how `app.css` implements the dark variant (`@custom-variant dark (&:is(.dark *))`)
2. **Content wrapper** — `AppContentLayout.vue` uses `rounded-[30px]` (too large) and `backdrop-blur-xl` (heavy). Step 2 brings it to `rounded-[26px]` and `backdrop-blur-sm` to match the brand radius and reduce GPU load

## What changed in each file

### `AppSidebarHeader.vue`
- **Added** `Sun` and `Moon` imports from `lucide-vue-next`
- **Added** `isDark` ref + `onMounted` that reads `document.documentElement.classList.contains('dark')` on load
- **Added** `toggleDark()` — toggles `.dark` class on `<html>` and writes `appearance` key to `localStorage`
- **Added** sun/moon `<button>` in the topbar right section, `size-8 rounded-md` matching the sidebar trigger
- Right side layout: `flex items-center gap-1.5` wrapping the toggle and the status indicator
- All existing props, breadcrumbs, and sidebar trigger are **unchanged**

### `AppContentLayout.vue`
- `rounded-[30px]` → `rounded-[26px]` (matches brand `--radius` scale used on cards)
- `backdrop-blur-xl` → `backdrop-blur-sm` (lighter GPU load, less visual noise)
- `shadow-[0_30px_85px_-52px_...]` → `shadow-[0_24px_60px_-44px_...]` (softer, matches Step 1 card style)
- Amber gradient top strip: `from-[#f6b56d]/14` → `from-[#f6b56d]/10`, height `h-24` → `h-20` (subtler)
- `dark:bg-white/[0.05]` → `dark:bg-white/[0.04]` (slightly more transparent in dark mode)
- `mt-5` → `mt-4 md:mt-5` (tighter spacing on mobile)
- `pb-5 md:pb-6` → `pb-6 md:pb-8` (more breathing room at bottom)
- Slot behaviour, `PageHeader`, and `hasHeader` logic are **unchanged**
