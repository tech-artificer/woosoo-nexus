# How to Add a New User

Staff members need a Woosoo Nexus account to access the admin dashboard. Each account is assigned a role that controls what they can see and do.

---

## Who Can Add Users

- **Super Admin** — can add users of any role, including other admins.
- **Admin** — can add users with Manager-level or below.

---

## Steps to Create a User

1. Click **Users** in the left sidebar navigation.
2. Click the **New User** button (top right of the page).
3. Fill in the user form:

| Field | Description |
|-------|-------------|
| **Name** | Full name of the staff member |
| **Email** | Work email — used to log in |
| **Password** | Minimum 8 characters. Combine letters, numbers, and symbols. |
| **Branch** | Assign the branch this user works at |
| **Role** | Select from the available roles (see below) |

4. Click **Create User**.

**Expected result:** The user appears in the Users list immediately. They can log in right away using the email and password you set.

---

## Roles Explained

| Role | Access Level |
|------|-------------|
| **Super Admin** | Full system access — all branches, all settings, event logs |
| **Admin** | Full access except super-admin-only functions |
| **Manager** | Orders, menus, reports, devices — no user/permission management |
| **Staff** | Order monitoring and service request handling only |

> **Best practice:** Assign the least-privileged role that lets the user do their job.

---

## Editing a User

1. Click **Users** in the sidebar.
2. Find the user in the list.
3. Click the **Edit** button (pencil icon) on their row.
4. Update the fields as needed (name, email, branch, role, or password).
5. Click **Save Changes**.

---

## Deactivating a User

If a staff member leaves, disable their account instead of deleting it (deactivated accounts cannot log in but their order history is preserved):

1. Open the user's edit page.
2. Toggle the **Active** switch to Off.
3. Click **Save Changes**.

---

## Troubleshooting

| Problem | Fix |
|---------|-----|
| Email already taken | Each email must be unique. Check if the user already exists. |
| User cannot log in after creation | Confirm the account is active and the email/password are correct. |
| Cannot see "New User" button | Your role may not have permission to create users. Contact your Super Admin. |
