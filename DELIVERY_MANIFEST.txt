QUICK START PATH (5 Steps, 10 Minutes)
================================================================================

Step 1: Configure
   Edit .env with your database details

Step 2: Deploy SQL Views
   mysql < krypton_woosoo_bi_views.sql (or psql for PostgreSQL)

Step 3: Build Docker Image
   docker build -f Dockerfile.processor.dhi -t bi-processor:dhi .

Step 4: Start Service
   docker-compose up -d bi-processor

Step 5: Generate Report
   docker exec bi-processor python bi_processor.py --report --output /app/reports/report.json

DONE. You now have:
   - Real-time order reconciliation views
   - Automated drift detection
   - Secure, hardened container
   - Daily automated reporting
   - BI tool connectivity

SECURITY SUMMARY (DHI Hardening)
================================================================================

Image Security Improvements:
   - Size: 250 MB -> 80 MB (68% reduction)
   - Privileges: Root (UID 0) -> Nonroot (UID 1000)
   - Attack surface: Full -> Minimal (no shell, no apt)
   - Compliance: CIS 4.1, 4.2, 6.6 + NIST 800-190 + PCI DSS 6.2

Container Security:
   - no-new-privileges:true - Can't escalate to root
   - Read-only root filesystem - Can be enabled
   - Resource limits - CPU 0.5, Memory 512 MB
   - Health checks - Validate dependencies on startup

Data Security:
   - Read-only views - No INSERT/UPDATE/DELETE
   - Separate user role - bi_readonly (SELECT only)
   - Audit trails - Created_at, detected_at timestamps
   - Zero source table modifications - Existing data untouched
