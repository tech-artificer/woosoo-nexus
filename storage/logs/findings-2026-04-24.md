## Critical Issues (Breaking)

### 1. Missing Database Tables — `cache` and `cache_locks`
- **Severity:** HIGH / Breaking
- **Affected Files:** laravel.log, scheduler.log, worker.log, reverb.log
- **Error:** `SQLSTATE[42S02]: Table 'woosoo.cache' doesn't exist`
- **Impact:**
  - Laravel cache driver completely non-functional
  - Scheduler mutex locking fails — scheduled tasks cannot acquire locks, risk of duplicate runs
  - Queue worker restart tracking broken (`illuminate:queue:restart` key unwritable)
  - KryptonContextService fails to load on every request (depends on cache)
  - PosConnectionService cannot apply POS connection from DB
- **Fix:** Run `php artisan migrate` to create cache tables

---

### 2. MySQL Authentication Failures — Docker Networking
- **Severity:** HIGH / Breaking
- **Affected Files:** laravel.log (460+ occurrences), scheduler.log, worker.log, reverb.log
- **Error:** `SQLSTATE[HY000] [1045] Access denied for user 'woosoo'@'172.18.0.4'`
- **Impact:**
  - All database-dependent features are intermittently or fully broken
  - App container at `172.18.0.4` cannot authenticate to MySQL
  - Cascades into every service that requires DB access
- **Fix:** Verify `DB_USERNAME`, `DB_PASSWORD`, `DB_HOST` in `.env` match MySQL container config

---

### 3. MySQL DNS / Connectivity Failure
- **Severity:** HIGH / Breaking
- **Affected Files:** reverb.log, laravel.log
- **Error:** `php_network_getaddresses: getaddrinfo for mysql failed: Try again`
- **Impact:**
  - Reverb WebSocket server cannot connect to DB on startup
  - Hostname `mysql` not resolving — MySQL container not in Docker network or not running
- **Fix:** Ensure MySQL container is running and on the same Docker network as the app
